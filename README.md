# QStory 2.4.1 破解版

这个作者和部分管理非常恶心，赞赏期一过就踢人，而且在以前付费的直接被吞掉？还有装什么正人君子，是君子会做这种模块么哈哈哈哈，随缘更新他们以后更新的版本哈哈

## 他做了什么？

QStory 是一个 QQ Xposed 模块，作者 **suzhelan** 在其中加入了以下恶意行为：

### 强制收费 & 功能锁
- 将基础功能锁在 **付费赞助** 墙后面，不赞助无法使用核心功能
- 多次弹出 **赞助弹窗**，每次打开设置页/恢复前台都会触发
- 设置 **"刷新用户信息"** 按钮，实时从服务器验证赞助状态

### 隐蔽的服务器通信
- 应用内置 5 个 API 接口（`/user-v2/doLogin`、`/user-v2/info`、`/user-v2/isLogin` 等），**持续向作者服务器发送用户数据**
- 后台静默通过多个线程 **上传 QQ 号、昵称、设备信息**
- 用户完全无感知，没有任何隐私提示

### 远程 Kill Switch
- 内置 **远程杀死开关**，作者可以随时通过服务器指令让你的模块停止工作
- 调用 `System.exit()` 强制终止进程

### 下载黑名单 + 利用管理权限踢人
- 从服务器 **下载黑名单列表**，包含作者想要针对的 QQ 号
- **滥用用户的群管理权限**，自动将黑名单中的人从用户管理的群中踢出
- 用户的管理员身份被模块 **劫持** 用于作者的私人恩怨？？
- 这就说明：**你装了这个模块，你的群就成了作者的打手工具**！！

### 代码深度混淆
- 所有类名使用 **中文字符混淆**（如 `叶花花落言飘`、`言言言言言花`）
- 所有关键字符串 **运行时加密解密**，故意阻止用户审计代码
- 明显是为了 **隐藏上述恶意行为**？？？

## 我们做了什么？

### 功能全免费
- **移除赞助弹窗**

### 彻底断网
- **禁用全部 5 个 API 接口**，不再向作者服务器发送任何数据
- 所有后台网络线程已短路（`return-void`），零流量消耗
- 你的 QQ 号、昵称、设备信息 **不再被上传**

### 移除 Kill Switch
- 远程杀死开关已用 `nop` 替换，作者无法远程控制你的模块
